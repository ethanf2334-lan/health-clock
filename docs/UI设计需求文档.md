# 健康时钟 UI 设计需求文档

## 1. 使用说明

本文档用于交给 UI 设计 AI、设计师或原型工具，生成一版更成熟的移动端 UI 设计方案。

建议同时提供当前 App 截图。文档负责说明产品语义、功能范围和设计约束；截图负责说明当前页面结构、信息量和已有问题。请不要只根据截图做“美化”，而要基于本文档重新梳理视觉语言、信息层级和组件系统。

## 2. 产品概述

产品名称：健康时钟

产品类型：iOS 优先的家庭健康提醒与健康档案管理 App

一句话定位：用 AI 帮助个人和家庭更轻松地记录复查、复诊、用药、体检、健康指标和医疗文档，形成一个清晰、可信、可持续使用的健康日历与档案系统。

核心价值：

- 一句话或语音即可创建健康提醒
- 上传体检报告、门诊记录、检查结果后，自动提取可确认的候选提醒
- 按家庭成员管理提醒、文档和健康指标
- 用健康日历帮助用户知道“今天该关注什么”“接下来有什么健康事项”

## 3. 目标用户

主要用户：

- 28-55 岁，有体检、复查、复诊或慢病管理需求的人
- 需要帮父母、配偶、孩子管理健康事项的家庭成员
- 希望用语音或一句话快速创建提醒，而不是填写复杂日历表单的人

典型场景：

- 用户输入“甲状腺三个月后复查”，系统生成一条复查提醒
- 用户上传体检报告，系统识别“6 个月后复查肺结节”等建议，并让用户确认加入日历
- 用户切换到“妈妈”成员，查看她的待办提醒、文档和指标记录
- 用户记录血压、血糖、体重、心率等健康指标，并查看历史趋势

## 4. 设计目标

这不是医疗诊断 App，也不是冷冰冰的数据看板。它应该像一个安静、可信、温柔但高效的家庭健康助手。

设计关键词：

- 清爽
- 可信
- 温和
- 有秩序
- 不焦虑
- 家庭感
- AI 友好但不过度科技化

希望达到的效果：

- 用户第一眼知道今天有没有重要健康事项
- 用户能快速切换家庭成员，不会记错人
- 健康提醒不只是普通日程，而是有类型、有优先级、有来源
- AI 创建提醒像聊天一样自然，但结果确认要清晰可靠
- 文档和 OCR 信息要像健康档案，而不是普通文件列表
- 健康指标要有趋势感，但不制造数据焦虑

## 5. 当前 UI 的主要问题

当前 App 功能已经比较完整，但 UI 仍不够成熟，主要问题包括：

- 页面视觉语言不统一，部分页面仍像默认 Material 组件堆叠
- 字体权重、标题层级和卡片密度容易显得拥挤
- 页面主色曾经过度偏绿色，缺少有节制的辅助色对比
- 首页“健康日历”还需要更强的第一屏设计，让用户快速理解今天状态
- AI 创建提醒面板需要更像一个自然输入工具，而不是普通表单
- 成员、文档、提醒、指标之间需要更一致的组件系统
- 文档详情页、指标页、成员档案页还需要更强的信息层级和视觉区分

## 6. 期望设计方向

### 6.1 整体风格

推荐方向：薄荷绿白为主，搭配少量蓝紫、珊瑚、暖黄、粉色等辅助色。

参考方向可以接近健康监测类 App 的轻盈感，但不要照搬任何单一产品。主色可以是健康、清新、可信的浅薄荷；辅助色用于表达 AI、提醒类型、健康指标类型、风险和待确认状态。

避免：

- 大面积深绿色
- 单一绿色调铺满所有页面
- 过重字体、过厚卡片、过多阴影
- 医院系统式蓝白表格感
- 过度可爱化，影响可信度
- 过度科技感或赛博风

### 6.2 色彩建议

主色：

- 薄荷绿 / 浅绿白，用于整体背景、健康状态、主要操作

辅助色：

- 蓝紫：AI、智能解析、语音输入
- 珊瑚：复诊、需要行动的提醒
- 暖黄：体检、待确认、轻提示
- 粉色：用药、风险、重要注意
- 蓝色：成员、家庭身份、文档信息

状态色：

- 成功：柔和绿色
- 警示：暖黄或珊瑚
- 错误/删除：红色只在危险操作中使用

### 6.3 字体与排版

中文界面要避免过度加粗。建议：

- 页面标题：清晰但不要过黑
- 卡片标题：中等字重
- 辅助文字：颜色更浅，行距舒适
- 避免多个大标题连续堆叠
- 列表和卡片内部要留足呼吸感

## 7. 信息架构

当前底部导航有 4 个 Tab：

1. 健康日历
2. 家庭成员
3. 健康档案
4. 我的

每个 Tab 的核心任务：

- 健康日历：查看今天和未来的健康提醒，快速创建提醒
- 家庭成员：管理家庭成员，切换当前成员，查看成员档案
- 健康档案：查看和上传医疗文档，查看 OCR/AI 提取信息
- 我的：账号、通知设置、健康指标入口、关于等设置

## 8. 核心页面设计需求

### 8.1 登录页

功能：

- 手机号验证码登录
- Apple 登录

设计目标：

- 可信、简洁、不要像普通后台登录页
- 突出“健康时钟”品牌感
- 可以加入轻量健康日历/时钟意象，但不要复杂插画堆叠

重点信息：

- App 名称
- 手机号输入
- 验证码输入
- 登录按钮
- Apple 登录按钮

### 8.2 健康日历首页

功能：

- 显示当前成员
- 展示今天或近期健康提醒状态
- 列表 / 周 / 月视图切换
- 今日 / 7 天 / 30 天 / 全部筛选
- AI 创建提醒底部入口
- 手动创建、记录指标、上传文档等快捷操作

设计目标：

- 第一屏要有明确的“今天健康事项状态”
- 重要提醒优先展示，详细列表向下滚动
- 不要让用户一进来就被列表淹没
- 可以使用情感化状态卡，但要贴合“健康日历”，不要直接照搬压力泡泡

可考虑设计：

- “今天状态”主卡片：如“今天节奏可控 / 今天需要多留意”
- 未来 7 天健康事项色块：用颜色和密度表示提醒数量与类型
- 提醒类型图标：复查、复诊、体检、用药、监测、自定义
- 待完成、已完成、逾期的视觉区分

### 8.3 AI 创建提醒面板

功能：

- 从健康日历底部展开
- 支持文字输入
- 支持语音输入
- 展示示例 chips
- 展示聊天记录
- AI 解析后生成待确认提醒卡片
- 用户可确认创建或进入表单修改

设计目标：

- 像自然聊天，不像复杂表单
- 但 AI 结果必须清晰可确认，避免误创建
- 示例要有帮助，但不要挤占输入区
- 面板高度要舒适，不能遮挡过多页面，也不能太矮

示例文案：

- 甲状腺三个月后复查
- 两周后复查肺结节 CT
- 每天晚上8点提醒妈妈吃药

### 8.4 家庭成员页

功能：

- 成员列表
- 添加成员
- 编辑成员
- 删除成员
- 设置当前成员
- 查看成员综合档案

设计目标：

- 不要出现重复标题
- 强调“当前成员”
- 成员卡片要亲切，但不能幼稚
- 需要清楚表达关系：本人、父亲、母亲、配偶、子女、其他

### 8.5 成员综合档案页

功能：

- 成员基础信息
- 待办提醒数量
- 文档数量
- 指标记录数量
- 最近动态
- 新建提醒、上传文档、记录指标快捷操作

设计目标：

- 更像一个健康档案首页，而不是普通详情页
- 统计卡片要轻，不要堆砌
- 最近动态要突出下一条提醒、最新文档、最新指标

### 8.6 文档列表页

功能：

- 按当前成员过滤文档
- 展示文档标题、分类、医院、日期、归属成员
- 上传文档
- 删除文档

设计目标：

- 医疗文档要有“档案感”和可信感
- 不要像普通文件管理器
- 分类标签和来源信息要清晰
- 空状态要引导用户上传报告

### 8.7 文档详情页

功能：

- 图片/PDF 预览
- 打开原文件
- 文档基本信息
- AI 提取信息
- OCR 识别全文
- 候选提醒列表
- 删除文档

设计目标：

- 信息层级非常重要
- AI 提取信息应该像“报告摘要”，不是 key-value 列表堆叠
- OCR 全文应折叠或弱化，避免占据主页面
- 候选提醒要突出“可行动”

### 8.8 文档上传页

功能：

- 选择成员
- 选择分类
- 输入标题、医院、日期
- 拍照 / 相册 / 文件
- 上传并 OCR

设计目标：

- 清楚表达流程：选择归属 → 添加信息 → 选择文件 → 上传识别
- 上传过程要有进度和安心反馈
- 文件选择区域可以做成更明显的上传卡片

### 8.9 OCR 审核页

功能：

- 展示 OCR / AI 解析结果
- 展示候选提醒
- 用户确认候选提醒
- AI 不可用时展示友好提示

设计目标：

- 重点突出“系统从报告中发现了什么”
- 候选提醒要像可确认任务
- 原始 OCR 文本可以弱化或折叠

### 8.10 健康指标页

功能：

- 切换成员
- 按指标类型筛选：血压、血糖、体重、心率、体温、血氧
- 指标历史列表
- 趋势图
- 新增记录

设计目标：

- 不要做成复杂医疗数据看板
- 趋势图应温和、低焦虑
- 指标类型需要固定颜色和图标
- 最新值、趋势、异常提醒要层级清楚

### 8.11 我的 / 设置页

功能：

- 显示账号状态
- 进入家庭成员、文档、健康指标、通知设置、关于
- 退出登录

设计目标：

- 设置页要安静、清晰
- 危险操作如退出登录要弱化但明确

### 8.12 通知设置页

功能：

- 权限状态
- 请求权限
- 待发送通知数量
- 测试通知
- 清空本地提醒
- 打开系统设置

设计目标：

- 让用户知道通知是否可用
- 避免技术化描述
- 操作层级要清楚，清空本地提醒属于风险操作

## 9. 必须保留的业务约束

- 每条提醒必须归属某个成员
- AI 生成提醒不能静默创建，必须让用户确认
- 用户必须能手动创建和编辑提醒
- 文档和健康指标都要按成员隔离
- 上传文档后 OCR/AI 结果可能失败，UI 要有友好降级
- 产品不提供医学诊断建议，只做提醒、归档和记录
- iOS 是首发平台，设计优先适配 iPhone
