# Flutter 前端开发进度

更新时间：2026-04-25

本文档用于说明 Flutter 客户端当前的实际完成情况。详细的跨前后端 MVP 对照，请参考 `docs/健康时钟-MVP项目进度总览.md`。

## 总体判断

当前前端已经具备 MVP 主链路，正在补齐体验和工程质量细节。

- 已完成：登录、成员管理、提醒管理、AI 创建提醒、文档上传/OCR、健康指标、通知调度、周/月日历视图
- 部分完成：个人中心、工程 lint 清理、Token 生命周期管理
- 未完成：语音输入、Apple 登录、Token 过期自动刷新

如果按“页面是否存在”来看，前端已经起步；
如果按“用户能否完成完整 MVP 主流程”来看，前端已经基本可体验。

## 当前主要问题

### 1. 主流程没有闭环

- Token 过期后目前需要重新登录
- 候选提醒确认体验仍可继续增强
- Apple 登录和语音输入尚未纳入当前实现

### 2. 页面存在但功能未接通

- 通知设置页未实现
- 关于页未实现

### 3. 工程收尾未完成

- `flutter test` 已通过
- `flutter analyze` 仍有既有 lint/info 和非阻断警告

## 建议的下一步

建议按以下顺序推进 Flutter MVP：

1. 完整端到端回归验证
2. 清理 `flutter analyze` 剩余 lint/info
3. 实现 Token 过期自动刷新
4. 评估 Apple 登录是否进入首版
5. 评估语音输入是否进入首版
