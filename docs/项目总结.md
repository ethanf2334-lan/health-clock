# 健康时钟 MVP 项目总结

## 项目概述

健康时钟是一款基于 AI 的家庭健康提醒与健康档案管理 App，采用 Flutter + FastAPI 技术栈，iOS 首发。

- 开发启动时间：2026-04-23
- 当前时间点：2026-04-26
- 当前状态：MVP 核心链路已基本可体验，AI 聊天式创建提醒、重复提醒通知、Apple 登录、语音输入、Token 续期、登录态恢复、OCR 候选提醒、成员综合档案、通知设置、关于页和首版 UI 设计系统已补齐
- 进度参考：详见 `docs/健康时钟-MVP项目进度总览.md`

## 当前结论

项目已经从“能力搭建”推进到“主流程可跑通”阶段：

- 后端核心能力完整度较高，登录、成员、提醒、文档/OCR、健康指标等 API 均已落地。
- 前端已接通主要业务链路，支持手机号验证码登录、成员管理、提醒管理、AI 创建提醒、文档上传/OCR、健康指标和本地通知。
- 第六轮补齐了健康日历的周/月网格视图，并修复了文档详情页导致 Flutter 测试无法编译的语法问题。
- 第七轮补齐了登录态持久化和 Token 自动续期，减少测试和日常使用时重复登录。
- 第八轮增强了 OCR 候选提醒确认能力，用户可从上传审核页或文档详情页把复查建议转成提醒，并根据常见日期表达自动预填时间。
- 第九轮将 Apple 登录和语音输入纳入首版，补齐 iOS 登录合规与语音创建提醒入口。
- 第十轮补齐通知设置页、关于页和成员综合档案页，剩余重点转为上线前真机回归。
- 第十一轮重构 AI 创建提醒为首页/日历页底部折叠式聊天输入，支持示例、聊天记录清空、原地确认卡片、当前成员创建和一键保存。
- 第十二轮补齐重复提醒闭环：AI 解析 daily/weekly/monthly 重复规则，本地通知按重复规则调度，列表和详情展示“每天/每周/每月”，并优化完成提醒为乐观更新。
- 第十三轮完成首版 UI 设计系统与首页/日历页样板改造：薄荷绿白主色、多辅助色功能点缀、安心状态主视觉、未来 7 天健康事项色块、AI 创建提醒面板溢出修复。
- 同步完成工程 lint 收尾：`flutter analyze` 已通过。

## 技术架构

### 前端

- 框架：Flutter 3.0+
- 状态管理：Riverpod
- 路由：GoRouter
- 网络请求：Dio
- 图表：fl_chart
- 文件选择：file_picker / image_picker
- 本地通知：flutter_local_notifications
- Apple 登录：sign_in_with_apple
- 语音输入：speech_to_text

### 后端

- 框架：FastAPI
- 认证：阿里云短信验证码 + Apple identity token 验证 + 自签 Supabase 兼容 JWT
- 数据库：Supabase Postgres
- 对象存储：Cloudflare R2
- AI 框架：LangChain
- AI 模型：Dashscope Qwen
- OCR：百度 OCR
