# 健康时钟 MVP 项目总结

## 项目概述

健康时钟是一款基于 AI 的家庭健康提醒与健康档案管理 App，采用 Flutter + FastAPI 技术栈，iOS 首发。

- 开发启动时间：2026-04-23
- 当前时间点：2026-04-24
- 当前状态：后端主链路已基本具备，前端处于“部分接通、尚未闭环”的阶段
- 进度参考：详见 `docs/健康时钟-MVP项目进度总览.md`

## 当前结论

项目已经完成了 MVP 的核心技术底座：

- FastAPI 后端、数据库迁移、RLS 权限策略已落地
- 成员管理、提醒管理、AI 文本解析、文档上传签名、OCR、健康指标记录等后端能力已具备
- Flutter 端已经完成应用骨架、成员列表/新增/删除、AI 输入与解析结果展示等基础页面

当前最大的差距不在后端，而在前端闭环：

- 登录流程尚未真正接入 Supabase
- AI 解析结果还不能直接创建提醒
- 提醒列表、手动创建提醒、日历视图仍未完成
- 文档上传、OCR 结果确认、候选提醒确认页面未实现
- 成员切换、健康指标页面、通知业务接入未完成

## 技术架构

### 前端

- 框架：Flutter 3.0+
- 状态管理：Riverpod
- 路由：GoRouter
- 本地数据库：Drift
- 网络请求：Dio
- 认证 SDK：Supabase Flutter

### 后端

- 框架：FastAPI
- 认证：Supabase Auth (JWT)
- 数据库：Supabase Postgres
- 对象存储：Cloudflare R2
- AI 框架：LangChain
- AI 模型：Dashscope Qwen
- OCR：百度 OCR

## 已实现能力

- JWT 鉴权与当前用户信息获取
