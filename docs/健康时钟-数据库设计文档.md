# 健康时钟数据库设计文档

## 1. 数据库选型

- 数据库：Supabase Postgres
- 版本：PostgreSQL 15+
- 扩展：pgvector（为后续 RAG 功能预留）

## 2. 表结构设计

### 2.1 users（用户表）

由 Supabase Auth 自动管理，无需手动创建。

关键字段：
- id (uuid)
- phone (text)
- email (text)
- created_at (timestamp)

### 2.2 profile_members（成员档案表）

```sql
CREATE TABLE profile_members (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  name VARCHAR(50) NOT NULL,
  relation VARCHAR(20),
  gender VARCHAR(10),
  birth_date DATE,
  height_cm DECIMAL(5,2),
  weight_kg DECIMAL(5,2),
  blood_type VARCHAR(10),
  chronic_conditions TEXT[],
  allergies TEXT[],
  notes TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  deleted_at TIMESTAMP WITH TIME ZONE
);

CREATE INDEX idx_profile_members_user_id ON profile_members(user_id);
CREATE INDEX idx_profile_members_deleted_at ON profile_members(deleted_at);
```

### 2.3 health_events（健康提醒表）

```sql
CREATE TABLE health_events (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  member_id UUID NOT NULL REFERENCES profile_members(id) ON DELETE CASCADE,
  title VARCHAR(200) NOT NULL,
  description TEXT,
  event_type VARCHAR(50) NOT NULL,
  scheduled_at TIMESTAMP WITH TIME ZONE NOT NULL,
  is_all_day BOOLEAN DEFAULT FALSE,
  repeat_rule JSONB,
  notify_offsets INTEGER[],
  status VARCHAR(20) DEFAULT 'pending',
  source_type VARCHAR(50) NOT NULL,
  source_text TEXT,
  ai_confidence DECIMAL(3,2),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  completed_at TIMESTAMP WITH TIME ZONE,
  deleted_at TIMESTAMP WITH TIME ZONE
);

CREATE INDEX idx_health_events_member_id ON health_events(member_id);
CREATE INDEX idx_health_events_scheduled_at ON health_events(scheduled_at);
CREATE INDEX idx_health_events_status ON health_events(status);
CREATE INDEX idx_health_events_deleted_at ON health_events(deleted_at);
```

**event_type 枚举值：**
- follow_up（复查）
- revisit（复诊）
- checkup（体检）
- medication（用药）
- monitoring（指标监测）
- custom（自定义）

**status 枚举值：**
- pending（待处理）
- completed（已完成）
- cancelled（已取消）

**source_type 枚举值：**
- ai_text（AI 文本创建）
- ai_voice（AI 语音创建）
- ai_document（AI 文档解析创建）
- manual（手动创建）

### 2.4 medical_documents（医疗文档表）

```sql
CREATE TABLE medical_documents (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  member_id UUID NOT NULL REFERENCES profile_members(id) ON DELETE CASCADE,
  file_url TEXT NOT NULL,
  storage_bucket VARCHAR(100) NOT NULL,
  storage_key VARCHAR(500) NOT NULL,
  file_name VARCHAR(255) NOT NULL,
  file_size BIGINT NOT NULL,
  mime_type VARCHAR(100) NOT NULL,
  category VARCHAR(50) NOT NULL,
  title VARCHAR(200),
  document_date DATE,
  hospital_name VARCHAR(200),
  ocr_text TEXT,
  ai_summary JSONB,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  deleted_at TIMESTAMP WITH TIME ZONE
);

CREATE INDEX idx_medical_documents_member_id ON medical_documents(member_id);
CREATE INDEX idx_medical_documents_category ON medical_documents(category);
CREATE INDEX idx_medical_documents_document_date ON medical_documents(document_date);
CREATE INDEX idx_medical_documents_deleted_at ON medical_documents(deleted_at);
```

**category 枚举值：**
- checkup_report（体检报告）
- examination_result（检查结果）
- outpatient_record（门诊病历）
- lab_report（检验单）
- prescription（处方/用药单）
- hospitalization_record（住院记录）
- other（其他）

### 2.5 health_metric_records（健康指标记录表）

```sql
CREATE TABLE health_metric_records (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  member_id UUID NOT NULL REFERENCES profile_members(id) ON DELETE CASCADE,
  metric_type VARCHAR(50) NOT NULL,
  value DECIMAL(10,2) NOT NULL,
  value_extra JSONB,
  unit VARCHAR(20) NOT NULL,
  recorded_at TIMESTAMP WITH TIME ZONE NOT NULL,
  note TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX idx_health_metric_records_member_id ON health_metric_records(member_id);
CREATE INDEX idx_health_metric_records_metric_type ON health_metric_records(metric_type);
CREATE INDEX idx_health_metric_records_recorded_at ON health_metric_records(recorded_at);
```

**metric_type 枚举值：**
- blood_pressure（血压，value_extra 存储 {systolic, diastolic}）
- blood_sugar（血糖）
- weight（体重）
- height（身高）
- heart_rate（心率）
- temperature（体温）
- blood_oxygen（血氧）

### 2.6 event_documents（提醒与文档关联表）

```sql
CREATE TABLE event_documents (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  event_id UUID NOT NULL REFERENCES health_events(id) ON DELETE CASCADE,
  document_id UUID NOT NULL REFERENCES medical_documents(id) ON DELETE CASCADE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE(event_id, document_id)
);

CREATE INDEX idx_event_documents_event_id ON event_documents(event_id);
CREATE INDEX idx_event_documents_document_id ON event_documents(document_id);
```

## 3. Row Level Security (RLS) 策略

### 3.1 profile_members

```sql
ALTER TABLE profile_members ENABLE ROW LEVEL SECURITY;

CREATE POLICY "用户只能访问自己的成员"
  ON profile_members
  FOR ALL
  USING (auth.uid() = user_id);
```

### 3.2 health_events

```sql
ALTER TABLE health_events ENABLE ROW LEVEL SECURITY;

CREATE POLICY "用户只能访问自己成员的提醒"
  ON health_events
  FOR ALL
  USING (
    member_id IN (
      SELECT id FROM profile_members WHERE user_id = auth.uid()
    )
  );
```

### 3.3 medical_documents

```sql
ALTER TABLE medical_documents ENABLE ROW LEVEL SECURITY;

CREATE POLICY "用户只能访问自己成员的文档"
  ON medical_documents
  FOR ALL
  USING (
    member_id IN (
      SELECT id FROM profile_members WHERE user_id = auth.uid()
    )
  );
```

### 3.4 health_metric_records

```sql
ALTER TABLE health_metric_records ENABLE ROW LEVEL SECURITY;

CREATE POLICY "用户只能访问自己成员的健康指标"
  ON health_metric_records
  FOR ALL
  USING (
    member_id IN (
      SELECT id FROM profile_members WHERE user_id = auth.uid()
    )
  );
```

### 3.5 event_documents

```sql
ALTER TABLE event_documents ENABLE ROW LEVEL SECURITY;

CREATE POLICY "用户只能访问自己的提醒文档关联"
  ON event_documents
  FOR ALL
  USING (
    event_id IN (
      SELECT he.id FROM health_events he
      JOIN profile_members pm ON he.member_id = pm.id
      WHERE pm.user_id = auth.uid()
    )
  );
```

## 4. 触发器

### 4.1 自动更新 updated_at

```sql
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_profile_members_updated_at
  BEFORE UPDATE ON profile_members
  FOR EACH ROW
  EXECUTE FUNCTION update_updated_at_column();

CREATE TRIGGER update_health_events_updated_at
  BEFORE UPDATE ON health_events
  FOR EACH ROW
  EXECUTE FUNCTION update_updated_at_column();

CREATE TRIGGER update_medical_documents_updated_at
  BEFORE UPDATE ON medical_documents
  FOR EACH ROW
  EXECUTE FUNCTION update_updated_at_column();
```

## 6. 数据迁移策略

- 使用 Supabase Migration 工具管理数据库版本
- 每次变更创建新的迁移文件
- 迁移文件命名：`{version}_{description}.sql`
- 支持回滚操作

## 7. 性能优化建议

- 为高频查询字段创建索引
- 使用 JSONB 存储非结构化数据
- 定期清理软删除数据（deleted_at 不为空的记录）
- 使用连接池管理数据库连接
