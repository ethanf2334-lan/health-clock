# 健康时钟开发指南

## 项目概述

健康时钟是一款基于 AI 的家庭健康提醒与健康档案管理 App，采用 Flutter + FastAPI 技术栈。

## 技术架构

### 前端（Flutter）
- **状态管理**: Riverpod
- **路由**: GoRouter
- **本地数据库**: Drift
- **网络请求**: Dio
- **认证**: Supabase Flutter SDK

### 后端（FastAPI）
- **认证**: Supabase Auth (JWT)
- **数据库**: Supabase Postgres
- **对象存储**: Cloudflare R2
- **AI 框架**: LangChain
- **AI 模型**: Dashscope Qwen3.6-Plus
- **OCR**: 腾讯云 OCR

## 开发环境搭建

### 1. 安装依赖

#### Flutter
```bash
# macOS
brew install --cask flutter

# 验证安装
flutter doctor
```

#### Python
```bash
# 创建虚拟环境
cd health-clock-backend
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### 2. 配置 Supabase

1. 访问 https://supabase.com 创建项目
2. 在 SQL Editor 中执行 `migrations/001_initial_schema.sql`
3. 在 Authentication 中配置登录方式：
   - 启用 Phone (SMS)
   - 启用 Apple (需要 Apple Developer 账号)
4. 获取项目 URL 和 API Keys

### 3. 配置 Cloudflare R2

1. 访问 Cloudflare Dashboard
2. 创建 R2 存储桶
3. 生成 API Token
4. 获取 Endpoint URL

### 4. 配置 AI 服务

#### Dashscope（阿里云百炼）
1. 访问 https://dashscope.aliyun.com
2. 创建 API Key
3. 选择模型：qwen-plus

#### 腾讯云 OCR
1. 访问腾讯云控制台
2. 开通 OCR 服务
3. 创建 API 密钥

### 5. 配置环境变量

#### 后端
```bash
cd health-clock-backend
cp .env.example .env
```

编辑 `.env` 填写配置：
```env
SUPABASE_URL=https://xxx.supabase.co
SUPABASE_ANON_KEY=eyJxxx...
SUPABASE_SERVICE_ROLE_KEY=eyJxxx...

R2_ACCESS_KEY_ID=xxx
R2_SECRET_ACCESS_KEY=xxx
R2_BUCKET=health-clock-files
R2_ENDPOINT_URL=https://xxx.r2.cloudflarestorage.com
R2_PUBLIC_URL=https://xxx.r2.dev

AI_API_KEY=sk-xxx
AI_MODEL_PROVIDER=dashscope
AI_MODEL_NAME=qwen-plus

TENCENT_SECRET_ID=AKIDxxx
TENCENT_SECRET_KEY=xxx
```

#### 前端
Flutter 使用 `--dart-define` 传递环境变量：
```bash
flutter run \
  --dart-define=SUPABASE_URL=https://xxx.supabase.co \
  --dart-define=SUPABASE_ANON_KEY=eyJxxx...
```

## 运行项目

### 后端
```bash
cd health-clock-backend
source venv/bin/activate
uvicorn app.main:app --reload
```

访问 http://localhost:8000/docs 查看 API 文档

### 前端
```bash
cd health-clock-app

# 初始化项目（首次运行）
flutter pub get
flutter pub run build_runner build --delete-conflicting-outputs

# 运行项目
flutter run
```

## 开发流程

### 1. 创建新功能

#### 后端
1. 在 `app/schemas/` 创建 Pydantic 模型
2. 在 `app/repositories/` 创建数据访问层
3. 在 `app/api/v1/` 创建路由
4. 更新 API 文档

1. 在 `lib/features/{feature}/` 创建功能模块
2. 创建 `data/` (数据层)、`domain/` (业务逻辑)、`presentation/` (UI)
3. 使用 Riverpod 管理状态
4. 使用 GoRouter 配置路由

### 2. 数据库变更

1. 在 `migrations/` 创建新的迁移文件
2. 在 Supabase SQL Editor 中执行
3. 更新数据库设计文档

### 3. 代码规范

- 使用 Black 格式化代码
- 使用 Flake8 检查代码质量
- 使用类型注解

#### Dart (前端)
- 遵循 Flutter 官方代码规范
- 使用 `flutter format` 格式化代码
- 使用 `flutter analyze` 检查代码

### 4. 测试

#### 后端
```bash
pytest tests/
```

#### 前端


## 常见问题

### 1. Supabase 认证失败
- 检查 token 是否过期
- 检查 SUPABASE_URL 和 SUPABASE_ANON_KEY 是否正确
- 检查用户是否已登录

### 2. R2 文件上传失败
- 检查 R2 配置是否正确
- 检查文件大小是否超过限制（10MB）
- 检查文件类型是否支持

### 3. AI 解析失败
- 检查 AI_API_KEY 是否正确
- 检查网络连接
- 查看错误日志

### 4. OCR 识别失败
- 检查腾讯云 API 密钥是否正确
- 检查图片 URL 是否可访问
- 检查图片格式是否支持

## 安全注意事项

1. **永远不要**提交 `.env` 文件到 git
2. **永远不要**在客户端代码中硬编码 API 密钥
3. 使用 HTTPS 传输数据
4. 定期更新依赖包
5. 使用 RLS 策略保护数据库
6. 使用预签名 URL 访问 R2 文件
