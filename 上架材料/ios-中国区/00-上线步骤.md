# 健康时钟 iOS 中国区上线步骤

更新时间：2026-05-09

> 适用身份：Apple Developer Program 个人开发者，中国区 App Store 首发。
> 当前项目：Flutter iOS App，Bundle ID：`com.lanxf.healthClock`，版本：`1.0.0+1`。

## 1. 上线前必须确认

- Apple Developer Program 已开通，账户为个人开发者，且能登录 App Store Connect。
- App Store Connect 中已接受所有有效协议；若未来要收费或内购，还需要补齐税务、银行与付费 App 协议。
- App 后端已部署到正式环境，iOS 构建时不要使用 `http://localhost:8000/api/v1`。
- 生产环境启用 HTTPS，并关闭 iOS `NSAllowsArbitraryLoads` 这类开发期网络放宽配置。
- 已准备隐私政策 URL。Apple 要求所有 App 提供隐私政策 URL。
- 如要在中国大陆 App Store 上架，准备 ICP/APP 备案信息。Apple 的 App 信息页会校验 ICP 备案号与工信部备案元数据是否匹配。
- App 内不要出现“诊断、治疗、处方、医学建议”等医疗结论式承诺；健康时钟应定位为“健康提醒与档案管理工具”。

## 2. Apple 开发者后台准备

1. 登录 Apple Developer。
2. Certificates, Identifiers & Profiles 中创建或确认 App ID。
3. Bundle ID 使用：`com.lanxf.healthClock`。
4. 按实际功能开启能力：
   - Sign in with Apple：如果正式版提供 Apple 登录，需要开启。
   - Push Notifications：如果未来使用远程推送再开启；当前本地通知不一定需要。
5. 配置开发和发布证书，或使用 Xcode 自动签名。

## 3. App Store Connect 创建 App

1. 登录 App Store Connect。
2. 进入“我的 App”，新建 App。
3. 平台选择 iOS。
4. 名称建议：`健康时钟`。
5. 主要语言建议：简体中文。
6. Bundle ID 选择：`com.lanxf.healthClock`。
7. SKU 建议：`health-clock-ios`。
8. 用户访问权限：默认“完整访问权限”。

## 4. 填写 App 信息

在 App Store Connect 中填入：

- 名称：见 `01-AppStoreConnect信息.md`
- 副标题：见 `01-AppStoreConnect信息.md`
- 类别：建议“医疗”或“健康健美”。如果担心医疗审核严格，可优先选择“健康健美”，但产品描述中仍要诚实说明文档和健康数据管理能力。
- 隐私政策 URL：使用你部署后的公开 URL。
- 年龄分级：一般可按低龄级别填写，但要如实回答“医疗/治疗信息”等问题。
- 价格与销售范围：选择中国大陆；如果 ICP/APP 备案暂未完成，建议先不要选择中国大陆，或准备好被提示“ICP Filing Number Missing”。
- 中国大陆可用性：填写有效 ICP/APP 备案号及相关支持文件（如适用）。

## 5. 准备产品页素材

- App 图标：1024 x 1024 PNG，无透明通道。
- 截图：至少 1 张，最多 10 张。建议首版准备 5 张 6.9 英寸竖屏截图，覆盖首页、AI 创建提醒、日历、文档识别、健康档案。
- App 预览视频：可选，15-30 秒，最多 3 个。
- 宣传文本：见 `01-AppStoreConnect信息.md`。

## 6. 归档上传构建

1. 在 `health-clock-app` 中执行：

   ```bash
   flutter clean
   flutter pub get
   flutter build ipa --release \
     --dart-define=API_BASE_URL=https://你的正式域名/api/v1 \
     --dart-define=SUPABASE_URL=https://你的-supabase-url \
     --dart-define=SUPABASE_ANON_KEY=你的-anon-key
   ```

2. 如使用 Xcode：
   - 打开 `health-clock-app/ios/Runner.xcworkspace`
   - 选择 Generic iOS Device 或 Any iOS Device
   - Product > Archive
   - Organizer 中 Distribute App > App Store Connect > Upload

3. 上传后等待 App Store Connect 处理构建。

## 7. TestFlight 内测

1. App Store Connect > TestFlight 选择构建。
2. 完成出口合规、加密、隐私相关问题。
3. 添加内部测试员，完成核心流程测试：
   - 登录/注册
   - 新增家庭成员
   - AI 文本创建提醒
   - 手动创建提醒
   - 本地通知权限
   - 上传图片/PDF 并完成 OCR
   - 健康指标录入
   - 账号/数据删除入口或客服删除流程

## 8. 提交审核

1. 选择已处理完成的构建。
2. 填写 App 隐私问答，参考 `01-AppStoreConnect信息.md`。
3. 填写审核备注和测试账号，参考 `04-审核备注与测试账号.md`。
4. 点击 Add for Review。
5. 在 Draft Submissions 中点击 Submit for Review。

## 9. 审核中与通过后

- 审核期间保持后端和测试账号可用。
- 如果被拒，优先看 Resolution Center 的具体条款，再按条款修复。
- 通过后可选择手动发布或自动发布。
- 上线后检查中国大陆产品页是否显示 ICP 备案号；若未备案或备案不匹配，可能无法在中国大陆销售。
